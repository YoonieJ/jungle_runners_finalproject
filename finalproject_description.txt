Jungle Runners is a MonoGame DesktopGL runner game where the player explores a jungle course, collects rewards, avoids hazards, follows recommended coin trails, and tries to clear three progressively harder stages. The project uses a local profile system so each player can enter a user id, save progress, keep best scores, earn stage stars, and preserve collected out-of-stage item ids between sessions.

The core gameplay is built around a scrolling three-row world. The runner stays near the left side of the screen while the stage moves toward the player. The player can move left and right between the front, middle, and back rows, jump or double jump over ground hazards, slide under projectile attacks, and spend collected rope items for a short speed and protection effect outside the boss fight. The row system also drives the front-view depth effect: closer rows draw lower and larger, while farther rows draw higher and smaller. Holding the top-view key switches to an overhead grid view that shows the same underlying stage layout from a more tactical perspective.

Each stage is generated from a stage definition with a name, description, and bronze, silver, and gold score thresholds. Stage 1, Overgrown Gate, introduces the basic runner flow with safer hazards and forgiving pickups. Stage 2, Serpent Causeway, increases the challenge with longer branches, more row changes, multi-row obstacles, meteors, and homing projectiles. Stage 3, Sunken Idol, is the final stage and ends with a boss encounter. In the boss fight, normal scrolling pauses, rope boosts are disabled, and the player must survive timed spear and boulder attacks by sliding or jumping at the correct moment.

The stage world is represented as a three-row grid of tiles. Tiles can hold coins, extra lives, shields, rope items, score boosters, collectible item placeholders, obstacles, meteors, straight projectiles, homing projectiles, branch markers, merge markers, and boss markers. After the basic content is generated, extra coins are placed like breadcrumbs along a recommended path so the player can read the intended route while running. The route graph is still a prototype: it currently supports approach, branch, and merge segments, but a fuller graph and route-selection system is planned.

The scoring system now uses accumulated distance points, accumulated coin points, and the stage 3 boss survival bonus. Score boosters activate a short x10 window instead of simply adding a flat score bonus. During that window, distance points earned while running and coins collected are multiplied by 10. At the end of each run, the game saves the user's stage progress, completion state, best score, top score per stage, star rating, remaining lives, score history, and any collected out-of-stage items. The save file is stored locally as JSON in the game's output folder.

The project includes sprite-based backgrounds, menus, player run frames, coins, shields, score boosters, meteors, projectiles, boss body art, and boss boulder attacks. The boss sprite has been enlarged and moved lower on screen so the text no longer blocks it. The current boss is a timed survival encounter, and future work will add boss movement and RPG-style dialogue before the boss stage.

The project is organized into folders for world data, factories, entities, components, screens, rendering, core systems, UI, utilities, data models, and content assets. The current playable prototype is primarily implemented through Game1 and Game1.Menu, while the surrounding screen, renderer, entity, and component classes show the intended modular architecture for continued development. This structure makes it possible to gradually move gameplay behavior out of the prototype class and into dedicated systems without changing the player-facing game loop.

Major implemented features include local user login, stage selection, a how-to-play screen, saved progress, scrolling jungle backgrounds, player animation frames, item and hazard sprites, two gameplay camera modes, prototype stage branching, recommended coin paths, pickups, shields, lives, x10 score boosters, damage feedback, temporary invulnerability, straight and homing map projectiles, rope use, and the stage 3 Sunken Idol boss encounter.

Planned improvements include moving all active gameplay into the screen classes, routing drawing through the renderer classes, replacing remaining proximity collision checks with precise hitboxes, fixing lane grounding so the player and hazards sit naturally on each lane, expanding the settings menu, adding boss movement, adding RPG-style stage and boss dialogue, adding more player animation frames plus jump/slide/rope sprites, implementing barricade sprites, adding more stages, replacing the hardcoded prototype graph with full route-selection logic, supporting character customization and out-of-stage item use, and showing a user scoreboard at the end of each round.
